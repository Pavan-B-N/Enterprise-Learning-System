# AZ-500: Microsoft Azure Security Technologies — Study Guide

## Exam Overview

| Detail | Info |
|--------|------|
| **Certification** | Microsoft Certified: Azure Security Engineer Associate |
| **Exam Code** | AZ-500 |
| **Duration** | 100 minutes |
| **Questions** | 40–60 |
| **Passing Score** | 700/1000 |
| **Cost** | $165 USD |
| **Renewal** | Annual |

## Skills Measured

### 1. Manage Identity and Access (25–30%)

- Configure Microsoft Entra ID security (MFA, Conditional Access, Identity Protection)
- Manage Privileged Identity Management (PIM)
- Configure application access (service principals, managed identities, app registrations)

### 2. Secure Networking (20–25%)

- Plan and implement network security (NSGs, ASGs, Azure Firewall)
- Plan and implement private access (Private Link, Private Endpoints, Service Endpoints)
- Plan and implement public access (DDoS Protection, WAF, Front Door)

### 3. Secure Compute, Storage, and Databases (20–25%)

- Configure advanced security for compute (VM security, AKS security, container security)
- Configure security for storage (encryption, SAS, network rules)
- Configure security for databases (TDE, Always Encrypted, dynamic data masking)

### 4. Manage Security Operations (25–30%)

- Configure and manage Microsoft Defender for Cloud
- Configure and manage Microsoft Sentinel (SIEM)
- Configure security monitoring (diagnostic settings, audit logs)
- Respond to security incidents (automated responses, playbooks)

## Study Strategy

1. **Security-first mindset**: Every decision should consider least privilege
2. **Microsoft Defender**: Know all Defender plans and their scope
3. **Sentinel**: Write basic KQL queries for threat hunting
4. **Hands-on**: Configure Conditional Access policies, network security

## Exam Tips

- Conditional Access policy conditions and grant/session controls are heavily tested
- Know the difference between Service Endpoints and Private Endpoints
- Microsoft Defender for Cloud plans: which resources each plan covers
- Sentinel: data connectors, analytics rules, playbooks (Logic Apps)
- Key Vault access policies vs RBAC — which to use when
