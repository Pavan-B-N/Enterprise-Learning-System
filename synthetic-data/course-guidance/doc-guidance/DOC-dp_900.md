## Skills Measured

### 1. Describe Core Data Concepts (25–30%)

- Describe data formats (structured, semi-structured, unstructured)
- Describe types of data workloads (transactional, analytical, streaming)
- Describe data analytics concepts (data warehousing, real-time analytics, data visualization)

### 2. Identify Considerations for Relational Data (20–25%)

- Describe relational concepts (tables, views, indexes, normalization)
- Describe Azure relational services (Azure SQL Database, Azure SQL MI, PostgreSQL, MySQL)
- Identify basic management tasks for relational data

### 3. Describe Considerations for Non-Relational Data (15–20%)

- Describe non-relational data models (document, key-value, column-family, graph)
- Describe Azure Cosmos DB APIs (Core SQL, MongoDB, Cassandra, Gremlin, Table)
- Describe Azure non-relational storage (Blob, Files, Tables)

### 4. Describe an Analytics Workload (25–30%)

- Describe analytics workloads (OLTP vs OLAP, ETL/ELT)
- Describe Azure analytics services (Synapse Analytics, Data Factory, Databricks)
- Describe data visualization (Power BI concepts: reports, dashboards, datasets)

## Study Strategy

1. **Conceptual**: This is a fundamentals exam — focus on "what" not "how"
2. **Comparisons**: Know when to use relational vs non-relational
3. **Azure services**: Match each data service to its use case
4. **Power BI**: Understand the basic components and workflow

## Exam Tips

- Know the difference between OLTP and OLAP workloads
- Cosmos DB consistency levels are commonly tested
- Understand ETL vs ELT — when to use each approach
- Power BI: reports vs dashboards vs datasets — simple but tested
- This exam does NOT require hands-on Azure experience
